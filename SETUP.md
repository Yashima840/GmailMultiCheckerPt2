# セットアップ手順(初回のみ・約15分)

この拡張機能は自分専用のGoogle Cloudプロジェクトを使います。
**この手順は最初に1回だけ**です。以後のアカウント追加は「設定画面 → アカウント追加 → Googleログイン」だけで完了します(アカウント数無制限)。

## 手順0: 拡張機能をChromeに読み込む

1. このフォルダで `npm install` → `npm run build` を実行(`dist/` が生成される)
2. Chromeで `chrome://extensions` を開く
3. 右上の「デベロッパーモード」をON
4. 「パッケージ化されていない拡張機能を読み込む」→ `dist` フォルダを選択
5. 拡張機能の「詳細」→「拡張機能のオプション」で設定画面を開き、
   **「リダイレクトURI」をコピー**しておく(手順3で使う)

> ⚠️ 拡張機能IDは `dist` フォルダの**絶対パス**から決まります。
> フォルダを移動・リネームするとIDが変わり、リダイレクトURIの再登録が必要になります。

## 手順1: Google Cloud プロジェクト作成 + Gmail API 有効化

1. https://console.cloud.google.com/ を開く(どのGoogleアカウントでもOK。管理用と考えて普段使いのものを推奨)
2. 上部のプロジェクト選択 →「新しいプロジェクト」→ 名前: `Gmail Multi Checker` → 作成
3. 左メニュー「APIとサービス」→「ライブラリ」→ `Gmail API` を検索 →「有効にする」

## 手順2: OAuth同意画面の設定

1. 「APIとサービス」→「OAuth同意画面」(新UIでは「Google Auth Platform」→「ブランディング」)
2. User Type: **外部(External)** を選択して作成
3. アプリ名: `Gmail Multi Checker`、サポートメール: 自分のメール、デベロッパー連絡先: 自分のメール → 保存
4. スコープの追加画面が出た場合は `.../auth/gmail.modify` と `.../auth/userinfo.email` を追加(スキップしても動作はする)
5. **重要**: 「公開ステータス」を「テスト」から **「本番環境」に変更**する
   (「アプリを公開」ボタン。確認ダイアログが出るがそのまま公開でOK)

> なぜ本番にするのか: テストのままだと**リフレッシュトークンが7日で失効**し、毎週全アカウントの再ログインが必要になります。
> 本番(未検証)だと、各アカウントの初回ログイン時に「Googleによる確認が完了していません」という警告が1回出るだけです。
> 自分のプロジェクト・自分のアカウントなので「詳細」→「Gmail Multi Checker(安全ではないページ)に移動」で問題ありません。

## 手順3: OAuthクライアントIDの作成

1. 「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」
2. アプリケーションの種類: **ウェブアプリケーション**
3. 名前: 任意(例: `chrome-extension`)
4. 「承認済みのリダイレクトURI」に、手順0でコピーした
   `https://<拡張機能ID>.chromiumapp.org/` を追加
5. 作成すると **クライアントID** と **クライアントシークレット** が表示されるのでコピー

## 手順4: 拡張機能に設定してアカウント追加

1. 拡張機能のオプション(設定)画面を開く
2. クライアントIDとクライアントシークレットを貼り付けて「設定を保存」
3. 「アカウント追加」→ Googleログイン → (未検証警告が出たら「詳細」→「移動」)→ 許可
4. 追加したいアカウントの数だけ手順3を繰り返す(2回目以降は警告と同意のみ)

以上で完了。ツールバーの封筒アイコンをクリックすると全アカウントの未読が表示されます。

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「redirect_uri_mismatch」エラー | 手順3のリダイレクトURIが設定画面の表示と完全一致しているか確認(末尾の `/` を含む) |
| 「アクセスをブロック: ○○はGoogleの審査を完了していません」で先に進めない | 手順2-5の「本番環境」への変更ができているか確認。テストのままの場合は「テストユーザー」に自分のメールを追加してもよい(ただし7日毎に再認証) |
| ポップアップに「認証が失効しています」 | 設定画面の「アカウント追加」から同じアカウントで再ログイン |
| 拡張機能を別フォルダに移した後ログインできない | 拡張機能IDが変わっています。設定画面の新しいリダイレクトURIをGoogle Cloudの認証情報に追加 |

  手順2-5の「本番環境への公開」がまだ反映されていない状態)。

  調査の裏付けとして、Gmailの制限付きスコープでも「本番(未検証)」に公開すれば認証画面で「詳細」→「(安全でないページに)移動」のバイパスが可能なことは確認できました。つ
  まり方針自体は正しく、公開ステータスだけが問題です。

  対処手順

  1. Google Cloud Console (https://console.cloud.google.com/) を開き、上部のプロジェクト選択が今回作った Gmail Multi Checker プロジェクトになっているか確認(エラー中の
  client_id: 568622664486-... が「APIとサービス」→「認証情報」に表示されるプロジェクトです)
  2. 左メニュー「APIとサービス」→「OAuth同意画面」を開く
    - 新UIの場合は「Google Auth Platform」→「対象(Audience)」
  3. 「公開ステータス」を確認してください。おそらく「テスト中(Testing)」になっています
  →「アプリを公開(PUBLISH APP)」ボタンを押して「本番環境(In
  production)」に変更。確認ダイアログで「検証が必要になる場合があります」という趣旨の警告が出ますが、そのまま公開してOKです(個人利用なので検証申請はしません)
  4. 数分待ってから、拡張機能の設定画面で再度「アカウント追加」
    - 今度は403ではなく「Googleによる確認が完了していません」という警告画面になるはずです
    - そこで「詳細」→「Gmail Multi Checker(安全ではないページ)に移動」→ アクセス許可、で完了です

  補足

  - 今すぐ動作だけ確認したい場合は、公開せずに「対象」画面の「テストユーザー」に自分のGmailアドレスを追加する方法でも認証は通ります。ただしテスト中のままだとリフレッ
  シュトークンが7日で失効して毎週再ログインになるため、最終的には手順3の本番公開をおすすめします
  - 本番公開後もまれに反映に時間がかかることがあります。5分ほど待っても同じエラーなら、Chromeで一度そのGoogleアカウントのログイン画面からやり直してみてください
